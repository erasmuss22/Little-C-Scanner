How to Download and use JLex and Java Cup

To use JLex and Java Cup on your own computer, you can download the latest versions from the Internet. However, the latest version of JLex doesn't allow comments in the second part of the specification. This is annoying, and will cause JLex to fail on the provided skeleton specification.

A good alternative is to use the jar files in

    ~cs536-1/public/JAVA
The files are called JLex.jar and java_cup.jar. To use them, copy them to your own computer and put them in the directory (folder) where you plan to write the code for your second programming assignment. You can either set your CLASSPATH to include the current directory as well as the two jar files, or you can use a classpath flag each time you use JLex to process your scanner specification and each time you compile the generated scanner.

Setting CLASSPATH

To set your CLASSPATH on a Linux machine, type:

    setenv CLASSPATH ".:JLex.jar:java_cup.jar"
Then (as long as you continue to work in the same window and the same directory) you can simply run JLex and compile the generated scanner as defined in the Makefile that was provided for program 2:
    java JLex.Main Little.jlex
    javac Little.jlex.java

Using the classpath flag

Linux

On a Linux machine, to use the classpath flag when you run JLex on your specification (in a file called Little.jlex), and then to compile the generated scanner, type the following, or put it into your Makefile:
    java -classpath .:JLex.jar:java_cup.jar JLex.Main Little.jlex
    javac -classpath .:JLex.jar:java_cup.jar Little.jlex.java

Windows

The CSL says that on Windows machines, the latest jdk does not add javac to the system path. So they suggest using something like the following commands, instead of the ones given for Linux users, above:
    java -classpath .;.\JLex.jar;.\java_cup.jar JLex.Main Little.jlex
    Program Files\java\jdk1.6.0_23\bin\javac.exe -classpath .;.\JLex.jar;.\java_cup.jar Little.jlex.java
